# Portable Operating Model — what transfers to any project

Distilled from `../operating_model/operating_model.md` and
`../operating_model/review_protocol.md`, which ground every step in the
`method-harness-compiler` git record (16 commits, f4c826f → 91b982b,
counted by script per DR-004: 15/16 carry `Co-Reviewed-By:` trailers —
5d96fd6 records its reviewer finding in prose as an in-body HONESTY NOTE
instead; 15/16 state a suite/test count — 2917804 is the exception; and
explicit-path staging assertions appear in the two commits where
concurrent flights shared the tree, 490470d and 510b79f).
Only the project-agnostic subset is restated here.

## The task loop

1. **Restate the task** before executing — say what you understood, against
   whatever plan or issue governs it (observed: every mhc commit subject
   names its milestone, e.g. `M2a / Phase 1` in 510b79f).
2. **Scope-check BEFORE execution** — declare what is allowed and what is
   forbidden this round; record friction with a frozen surface instead of
   editing it (510b79f: "zero installs, zero executions, zero signups";
   490470d excludes in-flight concurrent files by name).
3. **Plan with pins** — fix IDs, pass criteria, and forbidden edits in the
   work order before any execution or model call; pinned-ID discipline
   caught a misspelled workflow step a mechanical check would have silently
   resolved (operating_model.md Step 3; 5d96fd6 froze criteria pre-run).
4. **Minimal execution, deterministic first** — cheapest reliable method;
   on uncertainty emit an explicit TODO/stub that fails validation, never a
   guess (91b982b: "uncertainty always emits TODO_FILL stubs, never guesses").
5. **Layered review** — adversarial reviewers with ORTHOGONAL, non-overlapping
   lenses; one REQUEST_CHANGES outweighs the other lenses' APPROVEs; the
   catch is named in the commit body (review_protocol.md Layer 1).
6. **Independent re-verification** — never trust the report: re-run scripts,
   re-derive numbers by hand, live-check citations, and adjudicate agent
   conflicts by on-disk verification (490470d). **After any rename,
   grep the old value repo-wide** (f8740f8 "independent old-value grep";
   4437320 repo-wide grep + full suite after the project rename).
7. **Gated commit with explicit-path staging** — stage files by explicit
   path and assert staged count == expected count; never `git add -A` and
   hope (510b79f: "Explicit-path staging (6 files asserted)"). A review
   verdict gates the commit and the fired trigger is recorded in the body.
8. **CI watch to green** — the suite count is stated in the commit body and
   CI must go green after push; local green is not shipped green
   (review_protocol.md Layer 5: "Suite 638 -> 730 passing", 91b982b).

## Honest-failure doctrine

- **Publish your own failures** with the same prominence as wins: FAILs,
  UNSCORED counts, and defective runs ship in the report (1c4a77f README
  "we publish our failures"; 2917804 preserves the defective run).
- **Computed, not claimed** — numbers come from runners/scripts, never
  hand-asserted; on conflict the computed artifact wins (f8740f8: scorecards
  "regenerated by the runner only"; design-review finding C2 called
  self-assigned scores "the exact fabrication pattern").
- **A fix never lands in the change that benefits from it** — publish the
  divergence, fix next round (f8740f8: grader "deliberately NOT loosened in
  the change that would benefit from it"; v2 landed in 9532648).
- **Corrections are published, never concealed** — prepend a notice,
  recompute, keep the wrong version's existence on the record (2917804).

## No-fabrication rules

- No silent pass: skipped or inapplicable checks are RECORDED with a reason;
  unknown inputs yield UNSCORED, never a pass (f8740f8 grader registry).
- Never edit captured evidence; regrade the unchanged evidence through the
  tooling (CHANGELOG_STANDARD v0.7 via review_protocol.md Non-negotiable 4).
- A HALT, a NO_GO, or an honest refusal is a correct output, not an error
  to route around (portable_decision_rules DR-010).

## Explicitly EXCLUDED — do not apply outside mhc

These are real parts of the full harness but they are project machinery,
not portable discipline. In a non-mhc repo they do not exist and must not
be imitated blindly:

- **mhc phase gates and phase state** (`../operating_model/phase_gates.md`,
  `../context/L2_current_phase.md`) — another project's GO/NO_GO ladder.
- **mhc schemas and validators** (`../schemas/`) — the Person Harness
  Package standard's contract, frozen under its own changelog.
- **The package standard itself** (`../playbooks/phase0_static_package_standard.md`
  and the `phc` CLI conventions such as `.generated.yaml` drafts) — carry
  the draft-vs-governed *principle*, not the file-naming mechanism.

Everything above cites the source files or the mhc git log directly; if a
claim here cannot be traced to `../operating_model/` or a commit trailer,
treat it as drift and report it.
